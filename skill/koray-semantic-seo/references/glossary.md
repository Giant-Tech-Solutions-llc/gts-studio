# Glossary: Koray-style semantic SEO terms

Read this when the user or a task uses these terms and you want everyone aligned. Definitions
are operational — what the term means for the work, not just a dictionary line.

- **Topical authority** — the state where a search engine trusts a site as the source for an
  entire topic. The end goal; everything serves it.

- **Source context** — the site's identity: what it is, who it serves, what it's allowed to
  be authoritative about. The first input to a map; constrains what pages are credible.

- **Central entity** — the single entity the whole site (or section) orbits. Every page
  connects back to it.

- **Central search intent** — the dominant thing users come to accomplish. Threads through
  the whole map to keep coverage coherent.

- **Topical map** — the structured set of all pages, split into core and outer sections,
  each page a distinct contextual vector placed in a hierarchy.

- **Core section** — pages closest to the central entity and to conversion. Built first.

- **Outer section** — supporting / adjacent pages that build breadth and earn authority for
  the core section. Built second.

- **Contextual vector** — a page's unique combination of central entity + attribute + search
  intent. What makes it distinct. Duplicate vectors = cannibalization.

- **Contextual hierarchy** — the parent / child / sibling structure of the map. Nothing
  floats unattached.

- **Contextual flow** — the order in which context is built and connected, across the map
  and within a page. Internal links follow it.

- **Macro context** — the whole site's context. **Micro context** — an individual page's
  context. They must agree.

- **Triple** — a fact as subject → predicate → object ("creatine → increases → strength").
  The atomic unit of how information is structured.

- **Attribute** — a property of an entity (dosage, timing, cost, side effects). Covering all
  material attributes is what completeness means.

- **Named entity** — a specific recognized entity (a brand, a study, a standard) that adds
  precision and credibility.

- **Query semantics** — treating queries as surface forms of underlying intents and grouping
  them by intent, not string similarity.

- **Representative query** — the clearest expression of an intent; the question a section
  answers head-on.

- **Query template / phrase taxonomy** — the patterns queries take ("best X for Y", "X vs
  Z"), each often mapping to a section or page.

- **Information responsiveness** — how directly, quickly, and in-the-right-format content
  answers the actual question. First sentence answers the heading.

- **Cost of retrieval** — the effort a search engine spends to retrieve and trust an answer.
  Clarity, structure, and factual density lower it; fluff raises it.

- **Momentum / historical data** — authority built through consistent, focused publishing
  over time rather than sporadic posts.

- **Unique information gain** — information competitors don't provide (covered attributes,
  edge cases, corrections, original data). Parity is not authority.

- **Cannibalization** — two pages competing for the same intent / contextual vector,
  splitting signals. The most common structural error.

- **AEO (answer engine optimization)** — structuring content so AI answers, featured
  snippets, and LLM-driven discovery can extract and cite it. Served by the same
  answer-first, high-responsiveness structure.
